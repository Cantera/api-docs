22/**
23 * @defgroup phases Models of Phases of Matter
24 *
25 * These classes are used to represent the composition and state of a single
26 * phase of matter. Together these classes form the basis for describing the
27 * species and element compositions of a phase as well as the stoichiometry
28 * of each species, and for describing the current state of the phase. They do
29 * not in themselves contain Thermodynamic equation of state information.
30 * However, they do comprise all of the necessary background functionality to
31 * support thermodynamic calculations (see \ref thermoprops).
32 */
33
34//! Class Phase is the base class for phases of matter, managing the species and
35//! elements in a phase, as well as the independent variables of temperature,
36//! mass density (compressible substances) or pressure (incompressible
37//! substances), species mass/mole fraction, and other generalized forces and
38//! intrinsic properties (such as electric potential) that define the
39//! thermodynamic state.
40/*!
41 *
42 * Class Phase provides information about the elements and species in a
43 * phase - names, index numbers (location in arrays), atomic or molecular
44 * weights, etc. The set of elements must include all those that compose the
45 * species, but may include additional elements.
46 *
47 * It also stores an array of species molecular weights, which are used to
48 * convert between mole and mass representations of the composition. For
49 * efficiency in mass/mole conversion, the vector of mass fractions divided
50 * by molecular weight \f$ Y_k/M_k \f$ is also stored.
51 *
52 * Class Phase is not usually used directly. Its primary use is as a base class
53 * for class ThermoPhase. It is not generally necessary to overloaded any of
54 * class Phase's methods, which handles both compressible and incompressible
55 * phases. For incompressible phases, the density is replaced by the pressure
56 * as the independent variable, and can no longer be set directly. In this case,
57 * the density needs to be calculated from a suitable equation of state, and
58 * assigned to the object using the assignDensity() method. This also applies
59 * for nearly-incompressible phases or phases which utilize standard states
60 * based on a T and P, in which case they need to overload these functions too.
61 *
62 * Class Phase contains a number of utility functions that will set the state
63 * of the phase in its entirety, by first setting the composition, and then
64 * temperature and pressure. An example of this is the function
65 * Phase::setState_TPY(double t, double p, const double* y).
66 *
67 * For bulk (3-dimensional) phases, the mass density has units of kg/m^3, and the molar
68 * density and concentrations have units of kmol/m^3, and the units listed in the
69 * methods of the Phase class assume a bulk phase. However, for surface (2-dimensional)
70 * phases have units of kg/m^2 and kmol/m^2, respectively. And for edge (1-dimensional)
71 * phases, these units kg/m and kmol/m.
72 *
73 * Class Phase contains methods for saving and restoring the full internal state
74 * of a given phase. These are saveState() and restoreState(). These functions
75 * operate on a state vector, which by default uses the first two entries for
76 * temperature and density (compressible substances) or temperature and
77 * pressure (incompressible substances). If the substance is not pure in a
78 * thermodynamic sense (that is, it may contain multiple species), the state also
79 * contains nSpecies() entries that specify the composition by corresponding
80 * mass fractions. Default definitions can be overloaded by derived classes.
81 * For any phase, the native definition of its thermodynamic state is defined
82 * the method nativeState(), with the length of the state vector returned by
83 * by stateSize(). In addition, methods isPure() and isCompressible() provide
84 * information on the implementation of a Phase object.
85 *
86 * A species name is referred to via speciesName(), which is unique within a
87 * given phase. Note that within multiphase mixtures (MultiPhase()), both a
88 * phase name/index as well as species name are required to access information
89 * about a species in a particular phase. For surfaces, the species names are
90 * unique among the phases.
91 *
92 * @todo
93 * - Specify that the input mole, mass, and volume fraction vectors must sum
94 * to one on entry to the set state routines. Non-conforming mole/mass
95 * fraction vectors are not thermodynamically consistent. Moreover, unless
96 * we do this, the calculation of Jacobians will be altered whenever the
97 * treatment of non- conforming mole fractions is changed. Add setState
98 * functions corresponding to specifying mole numbers, which is actually
99 * what is being done (well one of the options, there are many) when non-
100 * conforming mole fractions are input. Note, we realize that most numerical
101 * Jacobian and some analytical Jacobians use non-conforming calculations.
102 * These can easily be changed to the set mole number setState functions.
103 *
104 * @ingroup phases
105 */

600 //! Elemental mass fraction of element m
601 /*!
602 * The elemental mass fraction \f$Z_{\mathrm{mass},m}\f$ of element \f$m\f$
603 * is defined as

606 * \f]
607 * with \f$a_{m,k}\f$ being the number of atoms of element \f$m\f$ in
608 * species \f$k\f$, \f$M_m\f$ the atomic weight of element \f$m\f$,
609 * \f$M_k\f$ the molecular weight of species \f$k\f$, and \f$Y_k\f$
610 * the mass fraction of species \f$k\f$.
611 *
612 * @param[in] m Index of the element within the phase. If m is outside
613 * the valid range, an exception will be thrown.
614 *
615 * @return the elemental mass fraction of element m.
616 */

619 //! Elemental mole fraction of element m
620 /*!
621 * The elemental mole fraction \f$Z_{\mathrm{mole},m}\f$ of element \f$m\f$
622 * is the number of atoms of element *m* divided by the total number of
623 * atoms. It is defined as:
624 *
625 * \f[
626 * Z_{\mathrm{mole},m} = \frac{\sum_k a_{m,k} X_k}
627 * {\sum_k \sum_j a_{j,k} X_k}
628 * \f]
