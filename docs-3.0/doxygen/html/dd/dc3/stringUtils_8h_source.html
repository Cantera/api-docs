68//! Translate a string into one doublereal value, with error checking
69/*!
70 * fpValueCheck is a wrapper around the C++ stringstream double parser. It
71 * does quite a bit more error checking than atof() or strtod(), and is quite
72 * a bit more restrictive.
73 *
74 * First it interprets both E, e, d, and D as exponents. stringstreams only
75 * interpret e or E as an exponent character.
76 *
77 * It only accepts a string as well formed if it consists as a single token.
78 * Multiple words will raise an exception. It will raise a CanteraError for
79 * NAN and inf entries as well, in contrast to atof() or strtod(). The user
80 * needs to know that a serious numerical issue has occurred.
81 *
82 * It does not accept hexadecimal numbers.
83 *
84 * It always use the C locale, regardless of any locale settings.
85 *
86 * @param val String representation of the number
87 * @returns a double
88 */
89doublereal fpValueCheck(const std::string& val);
